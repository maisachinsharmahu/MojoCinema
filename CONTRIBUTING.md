# Contributing to Mojocinema

First off, thank you for considering contributing to Mojocinema! This project is an open-source platform designed to bring premium movie discovery to everyone. Following these guidelines helps us keep the project organized and professional.

---

## 🚀 How to Contribute

### 1. Reporting Bugs
- Check the [Issues](https://github.com/maisachinsharmahu/MojoCinema/issues) to see if the bug has already been reported.
- If not, open a new issue.
- **Include**:
    - Steps to reproduce.
    - Expected vs. Actual behavior.
    - Screenshot/Video if applicable.
    - Flutter version (`flutter --version`).

### 2. Suggesting Enhancements
- Check current issues for similar suggestions.
- Be specific about what the enhancement adds to the user experience.
- If it's a major UI change, please include mockups.

### 3. Pull Requests
1. Fork the project.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes with clear messages (`git commit -m 'feat: add amazing feature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request.

---

## 🎨 UI & Design Policy (Crucial)

The **Glassmorphic Design** of Mojocinema is its core identity. To maintain this:
- Follow existing patterns in `lib/Pages/Home/home_page.dart` for transparency.
- Ensure all new widgets respect the established blur/glass hierarchy.
- **Mandatory**: Any distribution of the code includes the original design attribution to **Sachin Sharma**.

---

## 📐 Code Style Guidelines

- **Naming**: Use `CamelCase` for classes and `lowerCamelCase` for variables/functions.
- **Comments**: Write clear comments for complex logic, especially around API handling.
- **Null Safety**: Always use Dart null-safety features properly.
- **State Management**: Prefer Reactive patterns (ValueListenable) for UI updates.
